# VulnLab TS - プロジェクト設計方針

## コンセプト

**VulnLab TS** は「コードで説明できるセキュリティ教材」

### 目指す姿

```
「このPRで SQL Injection を潰しました。理由は〜」
をコードで語れる状態
```


## 技術スタック

| カテゴリ | 技術 | 理由 |
|----------|------|------|
| 言語 | TypeScript | 型安全性と学習コストのバランス |
| ランタイム | Node.js | フロント・バックエンド両方をカバー |
| パッケージ管理 | npm / pnpm | 標準的なエコシステム |
| テスト | Vitest | 高速で設定が簡単 |
| Webフレームワーク | Express / Hono | 軽量で脆弱性の再現がしやすい |
| DB | SQLite (better-sqlite3) | セットアップ不要、ファイルベース |

---


## 注意事項

### 倫理的ガイドライン

このリポジトリは**教育目的**で作成されています。

- 学んだ技術を許可なく他者のシステムに適用しないこと
- 実験は必ずローカル環境または許可された環境で行うこと
- 脆弱性を発見した場合は責任ある開示を行うこと

### 免責事項
